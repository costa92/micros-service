## Commit 规范


格式如下
  
  例：type(scope):本次提交概述 git commit -m "docs(README.md):添加编码规范"
  type: 本次 commit 的类型，诸如 bugfix docs style 等，参考如下:
  
    feat：添加新功能
    fix：修补缺陷
    docs：修改文档
    style：修改格式
    refactor：重构
    perf：优化
    test：增加测试
    chore：构建过程或辅助工具的变动
    revert：回滚到上一个版本
    merge：合并
    scope: 本次 commit 波及的范围

    subject: 简明扼要的阐述下本次 commit 的主旨，在原文中特意强调了几点：

      使用祈使句
      首字母不要大写
      结尾无需添加标点
